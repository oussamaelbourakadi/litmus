# Screenshots

The `.png` files here are **labeled placeholders**. Replace each with a real
screenshot of your running instance for a portfolio-ready README.

## How to capture

1. Start the stack: `docker compose up --build` (from the repo root).
2. Open the dashboard at `http://localhost:3000` and seed a little data
   (a project, a dataset with a few cases, and a couple of runs).
3. Take screenshots and save them here with these exact filenames (overwrite the
   placeholders):

| File | What it should show |
| --- | --- |
| `dashboard.png` | Projects list (or a project with its datasets). |
| `runs.png` | A run detail page — the metric cards and the per-case results table. |
| `compare.png` | A comparison of two runs, with regressed rows highlighted and the verdict badge. |
| `metrics.png` | A close-up of the metric cards, especially the success rate with its 95% confidence interval. |

## Tips

- Use a dark browser theme to match the dashboard.
- Keep a consistent window width (~1600px) so the images align in the README grid.
- On Windows: `Win+Shift+S` (Snipping Tool). macOS: `Cmd+Shift+4`. Linux: your
  screenshot tool of choice.
- Optional: record a short screen capture instead — see the "Record a 2-minute
  demo video" section in the top-level [`README.md`](../../README.md).
